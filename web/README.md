# QuantumAtlas Web

Vite + React 19 + TypeScript workspace for the QuantumAtlas web shell.

## Stack

- **Vite 7** — build / dev server
- **React 19**
- **TypeScript 5.6** with `@/*` path alias (`./src/*`)
- **Tailwind CSS v4** via `@tailwindcss/vite` (CSS-first config in `src/index.css`)
- **shadcn/ui** scaffolding (`components.json`, `lib/utils.ts` with `cn`) — components added on demand via `npx shadcn@latest add ...`
- **TanStack Router** (file-based routes under `src/routes/`)
- **TanStack Query** (data fetching hooks in `src/lib/queries.ts`)
- **lucide-react** for icons
- **marked + DOMPurify** for wiki markdown rendering
- **@hey-api/openapi-ts** for generating a typed API client from FastAPI's OpenAPI schema (run on demand)

## Layout

```
web/
├── components.json          # shadcn/ui config
├── vite.config.ts           # tailwind + tanstack-router + react plugins, @ alias
├── src/
│   ├── main.tsx             # RouterProvider + QueryClientProvider
│   ├── index.css            # @import "tailwindcss"
│   ├── App.css              # legacy hand-written design (still in use)
│   ├── routeTree.gen.ts     # auto-generated by TanStack Router (committed)
│   ├── lib/
│   │   ├── api.ts           # types + getJson + selectors
│   │   ├── queries.ts       # useQuery hooks
│   │   └── utils.ts         # cn() and small helpers
│   ├── components/          # Sidebar, Topbar, Panel, MetricGrid, ...
│   └── routes/              # __root, index, token, wiki/*, graph/*
```

## Routes

| URL | Route file |
|---|---|
| `/` | `routes/index.tsx` |
| `/token` | `routes/token.tsx` |
| `/wiki` | `routes/wiki.index.tsx` |
| `/wiki/search?q=...` | `routes/wiki.search.tsx` |
| `/wiki/page/<id>` (id may contain `/`) | `routes/wiki.page.$.tsx` |
| `/graph` | `routes/graph.index.tsx` |
| `/graph/node/<type>/<id>` | `routes/graph.node.$.tsx` |

These mirror the catch-all HTML routes in `atlas/server/main.py`, which all serve the same SPA shell.

## Common commands

```bash
npm install
npm run lint
npm run build              # tsr generate && tsc -b && vite build
cp -a dist/. ../atlas/server/static/web/   # publish to FastAPI's static dir
```

To regenerate the typed API client from a running FastAPI (`http://localhost:8000`):

```bash
npm run gen:api            # writes to src/lib/api/generated (configure in package.json or hey-api config)
```

## Notes

- `routeTree.gen.ts` is committed by convention so `tsc` can run without first running `tsr generate` in editors and CI.
- Tailwind v4 ships preflight in `@layer base`; existing rules in `App.css` are unlayered and therefore win without modification.
- shadcn/ui is initialized but no UI components have been imported yet; run `npx shadcn@latest add button` (etc.) when needed.
