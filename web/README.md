# QuantumAtlas Web

Vite + React 19 + TypeScript workspace for the QuantumAtlas web shell.

## Stack

- **Vite 7** ‚Äî build / dev server
- **React 19**
- **TypeScript 5.6** with `@/*` path alias (`./src/*` ‚Äî note: also in the root
  `tsconfig.json` so the shadcn CLI can resolve it)
- **Tailwind CSS v4** via `@tailwindcss/vite` (CSS-first config in `src/index.css`
  using `@theme inline` + shadcn light/dark CSS variables; `App.css` is gone)
- **shadcn/ui** (`components.json`, 17 components installed under
  `components/ui/`: alert, badge, button, card, dialog, dropdown-menu, input,
  label, scroll-area, select, separator, sheet, skeleton, sonner, tabs,
  textarea, tooltip)
- **TanStack Router** (file-based routes under `src/routes/`)
- **TanStack Query** (data fetching hooks in `src/lib/queries.ts`)
- **react-i18next + i18next-browser-languagedetector** ‚Äî URL-prefix
  internationalisation (`/zh/*`, `/en/*`, default `zh`)
- **next-themes** ‚Äî light/dark/system theme (storage key `qatlas_theme`)
- **lucide-react** for icons
- **marked + DOMPurify** for wiki markdown rendering
- **sonner** for toasts (mounted in `main.tsx`)

## Routes

Every user-facing route lives under a `/{lang}/` prefix (`zh` default, `en`
opt-in). The bare `/` redirects to `/{detectedLang}` based on (1) the
`qatlas_lang` localStorage key, (2) the browser `Accept-Language` header,
(3) `DEFAULT_LANG=zh`. Auth-only routes (`/login`, `/auth/callback`) stay
unprefixed.

| URL pattern | Route file |
|---|---|
| `/` | `routes/index.tsx` (redirect only) |
| `/login`, `/auth/callback` | `routes/login.tsx`, `routes/auth.callback.tsx` |
| `/{lang}` | `routes/$lang.index.tsx` (home) |
| `/{lang}/wiki` | `routes/$lang.wiki.index.tsx` |
| `/{lang}/wiki/search?q=...` | `routes/$lang.wiki.search.tsx` |
| `/{lang}/wiki/page/<id>` (id may contain `/`) | `routes/$lang.wiki.page.$.tsx` |
| `/{lang}/graph` | `routes/$lang.graph.index.tsx` |
| `/{lang}/graph/node/<type>/<id>` | `routes/$lang.graph.node.$.tsx` |
| `/{lang}/token` | `routes/$lang.token.tsx` |
| `/{lang}/pat` | `routes/$lang.pat.tsx` |

These all serve the same SPA shell mounted by the Go server's
`apis.Static` handler.

## Internationalisation

The URL prefix is the **authoritative** signal ‚Äî the `$lang` layout route
reads `:lang` from the route params and calls `i18n.changeLanguage()` on
every match. Localised language switching is just navigation: the
`LanguageSwitcher` component computes the equivalent path under the
target lang and navigates there with `router.navigate({ to, search })`,
preserving existing query string. No re-render games, no flash of stale
content.

- Bootstrap: `src/i18n/index.ts`
- Namespaces: `common home wiki graph token pat login auth`
- Strings: `src/i18n/locales/{zh,en}.json`
- React hook: `useTranslation('namespace')` or `<Trans i18nKey="..."
  components={{ ... }} />` for inline links
- Add a string ‚Üí edit both JSON files, no codegen step

## Theming

`next-themes` provides light / dark / system, persisted under
`qatlas_theme`. Colour tokens live in `src/index.css` under `:root` and
`.dark`, exposed to Tailwind via `@theme inline`. shadcn primitives use
those tokens directly (`bg-background`, `text-foreground`,
`border-border`, etc.).

## Common commands

```bash
npm install
npm run lint
npm run build              # tsr generate && tsc -b && vite build
npx shadcn@latest add <c>  # add another shadcn primitive
```

The build output (`dist/`) is embedded into the Go binary via
`internal/webui/spa.go` `//go:embed`; rebuild the Go server after a web
change is needed in production.

## Notes

- `routeTree.gen.ts` is committed by convention so `tsc` can run without
  first invoking `tsr generate` in editors / CI.
- All component file names are kebab-case to match shadcn convention.
- `@/*` alias must be configured in **both** `tsconfig.json` and
  `tsconfig.app.json` for the shadcn 4.x CLI to resolve target paths.
